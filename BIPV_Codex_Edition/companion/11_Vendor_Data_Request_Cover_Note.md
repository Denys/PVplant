# Vendor Data-Request Cover Note — iWin-Type BIPV v3

## Intended use

Send this with the questionnaire when you need a **single controlled technical request package** rather than ad hoc email exchanges.

## Draft

```text
Subject: Technical data request — glazing-integrated PV blind / iWin-type system

Please complete the attached questionnaire and response matrix for the exact product revision proposed for our evaluation.

Response rules:
1. Use revision-controlled documents only.
2. Mark each response as contractual or informational.
3. Where a value depends on configuration, state the valid range and the governing parameter.
4. Where a response is not applicable, explain why and provide the superseding rule or document.
5. Do not substitute marketing claims for drawings, datasheets, qualification reports, or procedures.

Priority A items and all knockout items are required for credible technical screening.

Requested return package:
- completed response matrix
- section drawing(s)
- electrical datasheet(s)
- controls/BMS point list
- qualification and reliability evidence
- commissioning and replacement procedures
- warranty and exclusions matrix

If any requested information is confidential, identify the NDA requirement and still provide an indexed placeholder so the gap remains visible in the review log.
```
