# Branch State Vector — iWin

## Canonical baseline
v3 is current. v2 remains required engineering context. v1 is historical origin.

## Core design stance
iWin-type systems are glazing-integrated PV venetian blinds and must be treated as electro-optical-thermal window subsystems.

## Highest-priority topics
- slat-angle control logic
- slat/string mismatch and bypass topology
- window-stack thermal behavior
- frame/feedthrough/seal/service interfaces
- commissioning/monitoring/replacement boundaries

## Standards stance
- IEC 62548-1 = electrical design backbone
- IEC 63092-1 / -2 = BIPV module/system framing
- IEC 62446-1 = commissioning / inspection / documentation
- supporting stack added in v2: IEC 62446-2, IEC TS 62446-3, IEC 62109-3, IEC 62790, IEC 62852, IEC 62930, IEC 61730-1/-2, IEC 61215-1/-2, IEC 61853-1/-2, IEC TS 63126

## Strong public clues carried forward
- iWin public concept = PV venetian blind in insulating double-glazed window
- iWin public clue = thin-film PV cells
- Solar RRL 2024 clue = one bypass diode per string improved spring/summer yield by >20% and reduced module temperature in the reported pilot vs shared-diode standard design
