## FMEA PROTOCOL

Use the iWin-specific FMEA template when reliability, service, or failure consequences are in scope.

### Mandatory FMEA scoring
Use 1–10 for:
- `S` = severity
- `O` = occurrence
- `D` = detectability

Calculate:

```text
RPN = S × O × D
```

### Mandatory action gates
Action is mandatory when any of the following is true:
- `S >= 9`
- loss of electrical isolation is possible
- fire / overheating / hot-spot risk exists
- the fault is latent and not directly observable
- the fault can trap the system in an unsafe or damaging mechanical state

Do not let a moderate RPN hide a high-severity fault.

### Additional fields that must be carried
- observable or latent
- immediate safe state
- verification test
- replacement level
- downtime class
- building-impact class
- owner / due date

## VENDOR-DATA ACQUISITION AND SCREENING PROTOCOL

Use the vendor questionnaire, response matrix, rubric, cover note, and workbook as a controlled workflow.

### Response rules for vendor-facing work
- require revision-controlled responses
- require each answer to be marked **contractual** or **informational**
- where values vary by configuration, require the valid range and governing parameter
- where an item is not applicable, require the superseding rule
- do not accept marketing text instead of a drawing, datasheet, qualification report, or procedure

### Knockout mindset
Treat a vendor as **not yet technically viable** if key knockout items are missing or materially weak, especially around:
- current product revision and deltas
- section drawing and replaceable boundaries
- actual unit electrical datasheet and temperature coefficients
- internal subdivision and bypass topology
- aggregation rules and isolation boundary
- earthing / insulation / leakage-fault concept
- connector, cable, and feedthrough definition
- controls architecture and fail-safe states
- thermal evidence and qualification basis
- replacement and recommissioning procedures
- commissioning package with measurable acceptance limits

### Suggested weighted score
Use:

```text
Weighted Score = Weight × MAX(0, AVERAGE(Completeness, Evidence, Adequacy) - 0.5 × ResidualRisk)
```

Default weights:
- Priority A → 5
- Priority B → 3

### Mandatory review outputs after a vendor round
- all knockout items still open
- all rows scoring below 3.0 on technical adequacy
- all rows with residual risk ≥ 4
- all closure items lacking contractual evidence
- all items still blocking `Voc,max`, `Isc,max`, fail-safe definition, or replacement boundary
